# reputation_hosts.txt — OFFLINE known-bad host/domain reputation feed.
#
# Bundled (embedded via go:embed) snapshot of known-bad C2 / exfil / dropper
# hosts. The iocscan matcher loads this list at init and flags a package whose
# source references one of these hosts — but ONLY as a strong signal when the
# reference is COUPLED with an actual import-time/install-time network send
# (reusing the existing netSend coupling). A bare reference is advisory.
#
# This is an OFFLINE snapshot. The matcher NEVER live-queries a feed at scan
# time — that would be a network dependency on the install hot path and a
# privacy leak (telling a third party which packages you scan). Refresh the
# snapshot out-of-band with scripts/detection-eval/ingest-reputation.sh.
#
# SOURCES (spine + tier-2), per the detection-lead roadmap:
#   1. OSSF malicious-packages — exfil/C2 domains extracted from the OSV
#      `database_specific` blocks of github.com/ossf/malicious-packages.
#   2. Self-curated C2 list — hosts confirmed in the DataDog 298-sample
#      malicious corpus (seeded below) and from incident response.
#   3. URLhaus (abuse.ch) — tier-2 bulk offline export. Fetch the CSV dump
#      out-of-band (https://urlhaus.abuse.ch/downloads/csv/) and run the
#      ingest script; do NOT live-query.
#
# FORMAT
#   One host per line. A host is a bare domain ("evil.tld"), a subdomain
#   ("c2.evil.tld"), or an IPv4 literal ("203.0.113.5"). Matching is
#   case-insensitive and suffix-aware: a feed entry "evil.tld" matches
#   "evil.tld" and "sub.evil.tld" but NOT "notevil.tld" (label boundary
#   enforced). Ports and paths in the source are stripped before matching.
#   Lines beginning with '#' and blank lines are ignored. Do NOT list
#   well-known CDNs / registries here — those are allowlisted in scan.go and
#   would be ignored anyway, but keeping the feed clean avoids confusion.
